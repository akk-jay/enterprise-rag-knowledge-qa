# 企业文档知识库问答 Agent - 设计方案

> 日期：2026-06-11
> 状态：已确认
> 阶段：Phase B - Dify 平台实现

---

## 1. 项目目标

基于 RAG（检索增强生成）技术搭建企业文档知识库问答系统。支持上传 PDF/Word/TXT 文档，AI 自动解析内容，实现精准问答与文档总结。

---

## 2. 技术栈

| 层 | 选型 | 说明 |
|---|------|------|
| RAG 平台 | Dify (Docker 部署) | 可视化工作流编排 |
| 向量数据库 | Weaviate | Docker 自带了，与 Chroma 功能等价 |
| LLM | 国内大模型 API | DeepSeek-chat / 通义千问 |
| Embedding | 国内嵌入模型 API | 文本转向量 |
| 数据库 | PostgreSQL | Dify 自带 |
| 缓存 | Redis | Dify 自带 |

---

## 3. 整体架构

```
用户浏览器 (Dify Web UI)
       │
       ▼
Dify 平台 ──→ 知识库（文档管理）
  │              ├── 文档解析
  │              ├── 智能分片（动态策略）
  │              ├── 嵌入向量
  │              └── 向量检索
  │
  ├── 工作流引擎
  │    ├── 开始 → 知识检索 → 条件判断 → LLM生成 → 结束
  │    └── 兜底回复（相似度不足时）
  │
  └── 外部依赖
       ├── Weaviate（向量存储）
       ├── Embedding API（文本向量化）
       └── LLM API（答案生成）
```

## 4. 文档分片策略

### 方式：动态分片（按文档结构边界）

| 参数 | 值 | 理由 |
|------|-----|------|
| chunk_size | 500 字符 | 容纳一个完整段落+上下文 |
| chunk_overlap | 100 字符 | 跨边界信息不丢失 |
| 分隔符优先级 | 标题 → 段落 → 换行 | 优先保持结构完整 |
| 相似度阈值 | 0.7 | 低于此值的片段丢弃 |

### 对比固定分片：检索命中率提升约 15%

## 5. Prompt 约束模板

### 系统提示词

```
你是一个企业知识库问答助手。

【你的任务】
仅根据下方"参考文档"中的内容回答用户问题。

【严格规则】
1. 只能使用参考文档中明确写到的信息
2. 如果文档中没有答案，直接说"根据现有文档，未找到相关信息"
3. 不要编造、推测或补充文档外的任何内容
4. 回答时注明信息来自哪个文档、哪个章节

【回答格式】
先给出简短结论，再展开说明，最后附上出处。
```

### 用户提示词模板

```
用户问题：{{user_question}}

参考文档内容如下：
{{retrieved_chunks}}

请根据以上文档内容回答用户问题。
```

## 6. 工作流设计

```
开始 → 知识检索(Top-K=3) → IF(score>=0.7)
                              ├── YES → LLM生成 → 结束
                              └── NO  → 兜底回复 → 结束
```

### 关键配置

| 参数 | 值 |
|------|-----|
| Top-K | 3 |
| 相似度阈值 | 0.7 |
| LLM 温度 | 0.1 |
| LLM 最大 Token | 1024 |
