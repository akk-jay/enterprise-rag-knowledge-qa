● 第五段：Dify 工作流编排设计

  ---
  什么是工作流？

  就是把"查文档→找资料→生成答案"串成一条自动流水线。 用户问一个问题，流水线自动跑完，输出答案。

  ---
  整体流水线（7 个节点）

  用户提问
     │
     ▼
  ┌──────────────┐
  │ ① 开始节点    │  接收用户输入的问题
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │ ② 知识检索    │  去向量库找最相关的文档片段（Top 3~5 条）
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │ ③ 相似度过滤  │  扔掉分数低于 0.7 的片段，全都不合格就提前结束
  └──────┬───────┘
         │
         ├── 有合格片段 ──→
         │                 │
         │                 ▼
         │         ┌──────────────┐
         │         │ ④ LLM 生成    │  把片段+问题+Prompt 发给大模型生成答案
         │         └──────┬───────┘
         │                │
         │                ▼
         │         ┌──────────────┐
         │         │ ⑤ 结束输出    │  返回答案给用户
         │         └──────────────┘
         │
         └── 无合格片段 ──→
                           │
                           ▼
                   ┌──────────────┐
                   │ ⑥ 兜底回复    │  "抱歉，未找到相关信息"
                   └──────┬───────┘
                          │
                          ▼
                   ┌──────────────┐
                   │ ⑤ 结束输出    │
                   └──────────────┘

  ---
  每个节点做什么——用人话版

  ┌────────────┬─────────────────────┬───────────────────────────────────────────────┐
  │    节点    │  在 Dify 里叫什么   │                    干什么                     │
  ├────────────┼─────────────────────┼───────────────────────────────────────────────┤
  │ ① 开始     │ Start               │ 接收用户问题，定义输入变量                    │
  ├────────────┼─────────────────────┼───────────────────────────────────────────────┤
  │ ② 知识检索 │ Knowledge Retrieval │ 指定用哪个知识库，返回最匹配的 Top-K 条片段   │
  ├────────────┼─────────────────────┼───────────────────────────────────────────────┤
  │ ③ 条件判断 │ IF/ELSE             │ 检查检索结果分数，高于 0.7 走上面，否则走下面 │
  ├────────────┼─────────────────────┼───────────────────────────────────────────────┤
  │ ④ LLM      │ LLM                 │ 填入 Prompt 模板，调用大模型生成答案          │
  ├────────────┼─────────────────────┼───────────────────────────────────────────────┤
  │ ⑤ 结束     │ End                 │ 返回最终结果                                  │
  ├────────────┼─────────────────────┼───────────────────────────────────────────────┤
  │ ⑥ 兜底回复 │ Answer              │ 当找不到相关文档时，固定回复一句话            │
  └────────────┴─────────────────────┴───────────────────────────────────────────────┘

  ---
  关键配置参数

  ┌────────────────┬────────┬─────────────────────────────────────────────┐
  │      参数      │ 填什么 │                   为什么                    │
  ├────────────────┼────────┼─────────────────────────────────────────────┤
  │ Top-K          │ 3      │ 取最相关的 3 个文档片段，够了；多了反而干扰 │
  ├────────────────┼────────┼─────────────────────────────────────────────┤
  │ 相似度阈值     │ 0.7    │ 在 IF 节点判断，score >= 0.7 才采用         │
  ├────────────────┼────────┼─────────────────────────────────────────────┤
  │ LLM 温度       │ 0.1    │ 接近 0，让回答稳定、少发挥、只照文档说      │
  ├────────────────┼────────┼─────────────────────────────────────────────┤
  │ LLM 最大 Token │ 1024   │ 留足够空间输出完整答案+出处                 │
  └────────────────┴────────┴─────────────────────────────────────────────┘

  ---
  为什么 Top-K 不是越多越好？

  Top-K = 1  →  信息不够全，可能漏掉关键内容
  Top-K = 3  →  刚好，够用又不冗余  ⭐
  Top-K = 10 →  太多无关信息混进去，AI 容易被带偏

  ---
  一条真实问答走一遍流水线

  用户问："迟到会怎么处罚？"
          │
          ▼
  知识检索：在向量库中找到 3 个片段：
    片段A（0.92分）："员工迟到30分钟以内，扣绩效分1分..."  ← 高度相关
    片段B（0.81分）："迟到超过1小时按旷工半天处理..."      ← 相关
    片段C（0.45分）："公司提供免费午餐..."                ← 无关
          │
          ▼
  条件判断：片段C低于0.7，丢弃。
          A(0.92) + B(0.81) 合格，继续 → LLM
          │
          ▼
  LLM 生成：
    "根据《员工手册》考勤制度章节：
     迟到30分钟以内扣绩效分1分；
     迟到超过1小时按旷工半天处理。"
          │
          ▼
  返回给用户 ✅
