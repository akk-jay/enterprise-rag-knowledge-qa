分片策略——用"切菜"来理解

  ---
  一句话：把一篇长文档切成小块，每块保持意思完整，然后存进数据库。

  ---
  为什么切？

  LLM 一次能读的字数有限，你不能把整本员工手册扔给它。而且相关内容越精炼，回答越准确。

  ---
  两种切法

  笨办法——固定分片：

  不管内容，每 500 字咔嚓一刀。

  原文：
  "…年假申请流程如下：第一，提前一周提交申请单；第二，
  ——咔嚓，断了——
  部门经理审批通过后，方可休假。"

  检索"怎么请年假？"→ 搜到上半段，流程不完整 → AI 回答也缺胳膊少腿

  聪明办法——动态分片：

  看文档的结构来切：标题、段落、章节。

  原文：
  [一、考勤制度] ← 碰到标题，从这里开始
    1.1 工作时间...（一整段完整保留）
    1.2 迟到处理...（一整段完整保留）
  [二、休假政策] ← 碰到标题，开始新 chunk
    2.1 年假...（一整段完整保留）

  每个 chunk = 一个意思完整的段落组

  ---
  用一张图对比

  固定分片：
  │──── chunk1 ────│──── chunk2 ────│──── chunk3 ────│
  │ "年假申请需提前" │ "审批通过可休假" │ "迟到扣绩效分1" │
  │                 │ ← 两个 chunk 把一句话拆散了

  动态分片：
  │────── chunk1 ──────│──────── chunk2 ─────────│
  │ [休假政策] 年假5天， │ [考勤制度] 迟到30分钟内，  │
  │ 申请需提前1周提交…   │ 扣当日绩效分1分…         │
  │ ← 每个 chunk 意思完整                             │

  ---
  三个参数用人话解释

  ┌────────────┬────────────────────┬────────────┬─────────────────────────────────────────┐
  │    参数    │        人话        │ 我们设多少 │                  类比                   │
  ├────────────┼────────────────────┼────────────┼─────────────────────────────────────────┤
  │ chunk_size │ 每块多大           │ 500 字     │ 一口能吃下的量                          │
  ├────────────┼────────────────────┼────────────┼─────────────────────────────────────────┤
  │ overlap    │ 相邻两块重叠多少   │ 100 字     │ 撕纸条时留个茬，拼回去不容易漏          │
  ├────────────┼────────────────────┼────────────┼─────────────────────────────────────────┤
  │ 相似度阈值 │ 相关性多高才拿来用 │ 0.7（70%） │ 低于 70 分的文档片段直接扔掉，不喂给 AI │
  └────────────┴────────────────────┴────────────┴─────────────────────────────────────────┘

  ---
  整个过程类比

  做一道菜"回答用户问题"

  1. 买菜     →  上传文档
  2. 洗菜切菜  →  文档解析 + 分片（这一步！）
     笨办法 = 乱刀切，大小一样但碎渣多
     聪明办法 = 顺着纹理切，块块完整
  3. 放冰箱   →  嵌入向量存入数据库
  4. 找食材   →  检索相关片段
  5. 下锅炒   →  把片段+问题发给 LLM 生成答案
