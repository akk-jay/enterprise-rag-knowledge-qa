# RAG 学习笔记

## 什么是 RAG？

RAG = Retrieval Augmented Generation = 检索增强生成

## 核心流程

1. 文档上传 → 2. 解析 → 3. 分片 → 4. 嵌入 → 5. 存储 → 6. 检索 → 7. 生成

## 关键概念

| 概念 | 说明 |
|------|------|
| Embedding | 把文本转成向量（一串数字） |
| 向量数据库 | 存储和检索向量的专用数据库 |
| 相似度 | 两个向量在数学上的接近程度 |
| Chunk | 文档切分后的小块 |
| Top-K | 一次检索返回最相关的 K 条结果 |
| Hallucination | AI 编造不存在的信息（幻觉） |

## 关键参数回顾

| 参数 | 我们的设置 | 作用 |
|------|-----------|------|
| chunk_size | 500 | 每块文本大小 |
| chunk_overlap | 100 | 块间重叠大小 |
| 相似度阈值 | 0.7 | 过滤低相关片段 |
| Top-K | 3 | 检索返回条数 |
| Temperature | 0.1 | 控制生成随机性 |
